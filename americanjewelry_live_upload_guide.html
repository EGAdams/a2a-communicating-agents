Quick Publish (single file)

# Local file example
LOCAL_FILE="/path/to/status.html"

# Upload
scp -o StrictHostKeyChecking=accept-new "$LOCAL_FILE" \
  tinman72@americansjewelry.com:public_html/status.html

# Verify on server
ssh -o StrictHostKeyChecking=accept-new tinman72@americansjewelry.com \
  "ls -l public_html/status.html && echo URL=https://americansjewelry.com/status.html"

Recommended Safe Workflow

  1. Back up current live file first.
  2. Upload new file.
  3. Check URL in browser (hard refresh).
  4. If bad deploy, restore backup.
# 1) Backup live file
ssh tinman72@americansjewelry.com \
  "cp public_html/status.html public_html/status.html.bak.$(date +%Y%m%d-%H%M%S)"

# 2) Upload new file
scp /path/to/status.html tinman72@americansjewelry.com:public_html/status.html

# 3) Verify
ssh tinman72@americansjewelry.com "ls -l public_html/status.html"

# 4) Rollback example (if needed)
ssh tinman72@americansjewelry.com \
  "cp public_html/status.html.bak.YYYYMMDD-HHMMSS public_html/status.html"

Troubleshooting

  • Permission denied: confirm correct SSH key/user and server access.
  • Page not updating: hard refresh browser (Ctrl+F5) and verify file timestamp on server.
  • Wrong path: ensure target is public_html/, not home dir root.
  • Success check: open https://americansjewelry.com/status.html and confirm latest timestamp/content.
